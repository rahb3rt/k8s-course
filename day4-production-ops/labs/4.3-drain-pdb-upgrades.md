# Lab 4.3 — Node maintenance: cordon, drain, PDBs & upgrades

## The three verbs
- **`kubectl cordon <node>`** — mark unschedulable. New pods stay away; existing pods untouched.
- **`kubectl drain <node> --ignore-daemonsets --delete-emptydir-data`** — evict all pods so they
  reschedule elsewhere (respects graceful termination + PDBs). DaemonSet pods are skipped.
- **PodDisruptionBudget** — availability contract honored by drains/rolling updates.

## The PDB-blocks-drain deadlock (demonstrated)
```bash
kubectl apply -f - <<'EOF'
apiVersion: policy/v1
kind: PodDisruptionBudget
metadata: {name: web-pdb}
spec:
  minAvailable: 2
  selector: {matchLabels: {app: web}}
EOF
kubectl drain lima-worker --ignore-daemonsets --delete-emptydir-data --timeout=90s
#   -> "Cannot evict pod as it would violate the pod's disruption budget" (retries, then times out)
```
`web` has 2 replicas + `minAvailable: 2`, and this is the only worker → nowhere to reschedule →
the drain **correctly refuses** to break availability. In production with several nodes, the
scheduler places replacements elsewhere first, then the drain proceeds.

**Fix the deadlock (not by deleting the PDB in a panic):**
```bash
kubectl uncordon lima-worker      # let evicted pods (sts-*, provisioner) reschedule back
kubectl delete pdb web-pdb        # (lab cleanup)
```

## Design rules
- PDB **+ spare capacity + spread** (`topologySpreadConstraints` / anti-affinity across nodes)
  = safe rolling maintenance & upgrades.
- PDB **without** capacity/spread = a node that won't drain. "Why won't this node drain?" →
  check the PDB math against how many nodes can host the pods.

## The upgrade flow (kubeadm)
```bash
kubeadm upgrade plan
kubeadm upgrade apply v1.31.x        # control plane first
# then, per node:  drain -> upgrade kubeadm/kubelet packages -> uncordon
```
- **One minor version at a time** (no 1.30 -> 1.32 jumps).
- **kubelet may lag the apiserver by up to 3 minors, never lead it** (version skew policy).
- The per-node **drain is step one of every upgrade** — which is why a stuck PDB blocks
  upgrades, not just maintenance.
- Bonus: `kubeadm upgrade` renews all certs (see 4.2) — upgrading yearly rotates them for free.
