# Lab 4.2 — Certificate expiry & rotation

kubeadm leaf certs expire after **1 year**; the CAs last **~10 years**. A cluster left
un-upgraded for a year locks *itself* out: the apiserver serves an expired cert, TLS
handshakes fail, and `kubectl` returns `x509: certificate has expired or is not yet valid`.
You can't kubectl your way out — kubectl is what broke.

> Architect's insight: `kubeadm upgrade` renews all these certs as a side effect, so
> "upgrade at least yearly" **is** your rotation mechanism. Un-upgraded clusters are cert
> time-bombs. Monitor expiry (alert at 30d) regardless.

## The two tiers (and why rotation is safe)
- **Leaf certs (~1yr):** apiserver, apiserver-etcd-client, apiserver-kubelet-client, etcd-*,
  front-proxy-client, and the embedded kubeconfigs (admin/scheduler/controller-manager).
- **CAs (~10yr):** `ca`, `etcd-ca`, `front-proxy-ca` — the trust anchors everyone pins.

Rotation re-signs the leaves with the **same CAs**, so nothing else has to change. That's why
leaf rotation is routine and CA rotation is a coordinated ordeal.

## Check
```bash
sudo kubeadm certs check-expiration      # leaf certs ~364d, CAs ~9-10y
```

## Rotate
```bash
sudo cp -r /etc/kubernetes/pki /root/pki-backup-$(date +%s)   # always back up first
sudo kubeadm certs renew all                                  # reissue every leaf cert

# GOTCHA: the running control-plane pods loaded the OLD certs at startup and only re-read the
# files on restart. Bounce ALL static pods (etcd certs rotated too):
sudo mv /etc/kubernetes/manifests /etc/kubernetes/manifests.off && sleep 20 && \
sudo mv /etc/kubernetes/manifests.off /etc/kubernetes/manifests

# 'renew all' also rewrote admin.conf — refresh your kubeconfig or kubectl uses a stale cert:
sudo cp /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

sudo kubeadm certs check-expiration      # new issuance time; expiry = now + 1yr
kubectl get nodes                        # brief apiserver bounce, then Ready
```

## Gotchas
- **The renew is a no-op until you restart the static pods.** Files new, process old.
- The kubelet's own client cert (in `/var/lib/kubelet/pki`) auto-rotates and is **not** in this
  list — only the control-plane/kubeconfig certs need manual `renew all`.
- On a cluster only hours old, the expiry *year* won't change on renewal — the timestamp shift
  (time-of-day) is the proof it rotated.
