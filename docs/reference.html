Objects, workloads & architecture

Quick definitions and "when to use which" for the pieces you meet across the course. Each links to the day that dissects it.

Cluster architecture

Everything is a client of the API server, which is the only thing that talks to etcd. The control plane makes decisions; the nodes run the work. (Deep dive: Day 1.)

Control plane (cp)
kube-apiserverThe front door. Validates + serves every object; the only component that reads/writes etcd.
etcdThe database. Consistent key-value store holding all cluster state (as protobuf).
kube-schedulerAssigns each new Pod to a node (resources, affinity, taints, spread).
kube-controller-managerRuns the built-in reconcile loops (Deployment, ReplicaSet, Node, Job…).
Worker node
kubeletThe node agent. Runs the Pods the API server assigns; reports status. Not a Pod itself.
kube-proxyPrograms iptables/IPVS so Service virtual IPs route to Pod IPs.
containerd (CRI)The container runtime — pulls images, runs containers via runc.
PodsYour workloads. Wired to the network by the CNI plugin (Flannel here).

Workloads — when to use which

These are the objects that manage Pods. You almost never create a bare Pod. (Lifecycle: Day 3.)

ObjectWhat it managesReach for it when…
DeploymentStateless, interchangeable replicas (via a ReplicaSet)Web apps, APIs — anything where pods are cattle. Rolling updates & rollbacks.
ReplicaSetA fixed number of identical PodsRarely directly — a Deployment owns these and manages them for you.
StatefulSetPods with stable identity: ordinal name, own PVC, ordered startupDatabases & clustered apps (Kafka, etcd, Postgres) needing stable name/DNS/storage.
DaemonSetOne Pod per nodeNode agents: log shippers, node-exporter, the CNI, CSI, security agents.
JobRun-to-completion PodsBatch work: migrations, one-off processing. Retries until success.
CronJobJobs on a scheduleScheduled batch: backups, reports, cleanups.
PodOne or more co-located containers (the atom)Almost never alone — debugging/one-shots only. Let a controller own it.
Deployment vs StatefulSet

Cattle vs. named livestock. If any pod is interchangeable and holds no unique data → Deployment. If pod db-0 must always be db-0 with its disk → StatefulSet.

DaemonSet vs Deployment

"On every node" (DaemonSet) vs "N replicas wherever they fit" (Deployment). Node-level infra is a DaemonSet; apps are Deployments.

Networking objects

How traffic finds Pods. (Deep dive: Day 2.)

ObjectDefinition
Service · ClusterIPA stable internal virtual IP that load-balances to a set of Pods. The default. (iptables/conntrack under the hood.)
Service · NodePortExposes the Service on a fixed port on every node. Basic external access.
Service · LoadBalancerProvisions a cloud load balancer that points at the Service. Standard external access in the cloud.
Headless ServiceclusterIP: None — no VIP; instead one DNS record per Pod. Powers StatefulSet identity.
IngressL7 HTTP(S) routing (by host/path) to Services, via an ingress controller (nginx, Traefik…).
Gateway APIThe modern, more expressive successor to Ingress (roles-based, multi-protocol).
NetworkPolicyPod-level firewall (allow/deny by label/namespace). Only enforced if the CNI implements it — Flannel doesn't.
EndpointSliceThe live list of Pod IPs behind a Service — what kube-proxy programs and what a drain updates.

Config & storage

(Deep dive: Day 3.)

ObjectDefinition
ConfigMapNon-secret config (key/values, files) injected as env vars or mounted files.
SecretSensitive data. base64-encoded, not encrypted by default — enable encryption-at-rest / use External Secrets in prod.
PersistentVolume (PV)A real piece of storage in the cluster (disk, NFS share, cloud volume).
PersistentVolumeClaim (PVC)A request for storage ("1Gi, RWO"). Binds to a PV. Pods mount the claim, not the PV.
StorageClassA template + provisioner for creating PVs on demand (dynamic provisioning).
emptyDirEphemeral, Pod-scoped scratch space — gone when the Pod dies. (Not persistent!)

The request & packet flow

Creating a Pod: kubectl → API server (authn/authz/admission) → etcd
→ scheduler picks a node → kubelet on that node → containerd (CRI) → CNI wires the network → Pod Running
A request to a Service: client → ClusterIP (a virtual IP that exists only as an iptables rule)
→ kube-proxy's DNAT rewrites it to a real Pod IP → conntrack pins the choice → Pod
cross-node? the packet is VXLAN-encapsulated node→node and decapsulated on arrival.

Glossary

Pod
The smallest deployable unit — one or more containers sharing a network namespace + storage.
Namespace
A virtual cluster for scoping and isolating objects (e.g. kube-system, default).
Node
A machine (VM/physical) that runs Pods, managed by its kubelet.
Label / Selector
Key/value tags on objects; selectors match them. How Services find Pods and Deployments own ReplicaSets.
Taint / Toleration
A node repels Pods (taint) unless the Pod tolerates it. Keeps workloads off control-plane nodes.
Affinity / Anti-affinity
Attract or repel Pods relative to nodes or other Pods (spread replicas, co-locate).
Requests / Limits
Requests = guaranteed resources (used for scheduling); limits = hard caps (enforced by cgroups). Drive QoS.
QoS class
Guaranteed / Burstable / BestEffort — derived from requests==limits; sets OOM-kill priority.
HPA / VPA
Horizontal Pod Autoscaler scales replica count on metrics; Vertical scales a pod's requests/limits.
PodDisruptionBudget
"Never take me below N available" during voluntary disruptions (drains, rollouts). Honored by drain.
RBAC
Role/ClusterRole (permissions) + RoleBinding/ClusterRoleBinding (who gets them). Subjects: users, groups, ServiceAccounts.
ServiceAccount
An identity for Pods to call the API. Tokens are signed by sa.key (Day 1).
ResourceQuota / LimitRange
Per-namespace caps on total resources (Quota) and per-object defaults/bounds (LimitRange).
CRD
CustomResourceDefinition — adds your own API type (e.g. ReplicaPin, A1). Served like any built-in.
Operator / Controller
A reconcile loop that drives observed state → desired state for some (often custom) resource. (A1.)
kubeconfig / context
The file (~/.kube/config) telling kubectl which cluster + user + namespace to talk to.
Static / mirror pod
A Pod the kubelet runs from a file on disk (the control plane); the API copy is read-only. (Day 1.)
