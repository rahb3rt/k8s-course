Setup

Bootstrap the cluster from zero

A 2-node kubeadm cluster on an Apple Silicon Mac using Lima VMs.

0 · Prerequisites (Mac)

# Kubernetes is Linux — the Mac only HOSTS the Linux VMs.
brew install lima kubectl

1 · Create the VMs

Two Ubuntu 24.04 arm64 VMs on Apple’s hypervisor (vz), joined to Lima’s user-v2 network so they can reach each other.

limactl start --name=cp     --vm-type=vz --tty=false \
  --set '.cpus=2 | .memory="4GiB" | .disk="20GiB" | .networks=[{"lima":"user-v2"}]' \
  template://ubuntu-24.04
limactl start --name=worker --vm-type=vz --tty=false \
  --set '.cpus=2 | .memory="4GiB" | .disk="20GiB" | .networks=[{"lima":"user-v2"}]' \
  template://ubuntu-24.04
limactl shell cp -- ping -c3 192.168.104.3   # prove VM-to-VM connectivity

2 · Prepare both nodes

setup/node-prep.sh disables swap, loads kernel modules, sets the bridge sysctls, installs containerd (with the critical SystemdCgroup = true), and installs kube* 1.31.

limactl shell cp     -- sudo bash < node-prep.sh
limactl shell worker -- sudo bash < node-prep.sh
Two lines to actually read

SystemdCgroup = true (cgroup-driver mismatch = crash-looping kubelet) and bridge-nf-call-iptables (off = Services silently don’t route). Also installs conntrack/socat — kubeadm treats missing conntrack as a fatal preflight.

3 · Init the control plane (cp)

limactl shell cp -- sudo kubeadm init \
  --apiserver-advertise-address=192.168.104.1 \
  --pod-network-cidr=10.244.0.0/16 \
  --kubernetes-version=stable-1.31
  • advertise-address must be cp’s user-v2 IP — it’s baked into the apiserver cert SAN. Wrong value → TLS failures from the worker.
  • pod CIDR is Flannel’s default and disjoint from the node net (overlapping CIDRs = nasty routing bug).

Save the kubeadm join … line it prints.

4 · Wire up kubectl (cp)

mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
kubectl get nodes    # NotReady until CNI installed — expected

5 · Install the CNI (Flannel)

kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
kubectl get nodes -w   # cp flips to Ready in ~30s

6 · Join the worker

limactl shell worker -- sudo kubeadm join 192.168.104.1:6443 \
  --token <TOKEN> --discovery-token-ca-cert-hash sha256:<HASH>

Lifecycle

limactl stop  cp worker sandbox   # free RAM (state persists)
limactl start cp worker           # resume intact
