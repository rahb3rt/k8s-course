The Controller Pattern

The operator → architect crossing. Build a CRD + controller (Go + kubebuilder) that pins a Deployment's replica count and self-heals drift — the same loop the ReplicaSet controller and every operator runs.

The reconciliation loop

A controller is a loop that never ends: observe desired → observe actual → act to close the gap → repeat. Desired state = a spec you declare; observed = what's real; the controller drives observed toward desired.

Level-triggered, not edge-triggered

Edge = react to events ("a pod was deleted → make one"); miss the event and you stay broken. Level = react to state ("whatever desired says, make it so"). Miss an event? The next loop reconciles to the current truth anyway. That's why controllers heal after a crash — they depend on state, not on a history of events.

A1.1 · Feel the loop (15 lines of bash)

Declare desired state in a ConfigMap, then run the whole pattern by hand. Scale web wrong in another terminal and watch it self-heal.

kubectl create deployment web --image=nginx --replicas=2
kubectl create configmap desired-state --from-literal=web-replicas=3

while true; do
  desired=$(kubectl get cm desired-state -o jsonpath='{.data.web-replicas}' 2>/dev/null)
  actual=$(kubectl get deployment web -o jsonpath='{.spec.replicas}' 2>/dev/null)
  if [ -n "$desired" ] && [ "$desired" != "$actual" ]; then
    echo "$(date +%T)  reconcile: web $actual -> $desired"
    kubectl scale deployment web --replicas="$desired"
  fi
  sleep 5      # the RequeueAfter of the shell world (level-triggered resync)
done

A1.2 · Scaffold a real operator (Go + kubebuilder)

On cp. Work in the writable guest home ~ — not the read-only /Users/… Mac mount.

# toolchain
GOVER=$(curl -sL https://go.dev/VERSION?m=text | head -1)
curl -sL https://go.dev/dl/${GOVER}.linux-arm64.tar.gz | sudo tar -C /usr/local -xzf -
echo 'export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin' >> ~/.bashrc && source ~/.bashrc
cd ~ && curl -L -o kubebuilder https://go.kubebuilder.io/dl/latest/linux/arm64
chmod +x kubebuilder && sudo mv kubebuilder /usr/local/bin/
sudo apt-get install -y make build-essential        # kubebuilder is Makefile-driven

# scaffold — group is 'ops', NOT 'apps' (built-in, would collide)
mkdir -p ~/replicapin-operator && cd ~/replicapin-operator
kubebuilder init --domain quarx.co --repo quarx.co/replicapin
kubebuilder create api --group ops --version v1 --kind ReplicaPin --resource --controller

A1.3 · Design the Spec + write Reconcile

In api/v1/replicapin_types.go, replace the placeholder Foo with real desired-state fields — the +kubebuilder markers become OpenAPI validation in the generated CRD:

type ReplicaPinSpec struct {
	// +required
	DeploymentName string `json:"deploymentName"`

	// +kubebuilder:validation:Minimum=0
	// +required
	Replicas int32 `json:"replicas"`
}

Then the reconcile loop in internal/controller/replicapin_controller.go — your bash loop, typed and hardened:

func (r *ReplicaPinReconciler) Reconcile(ctx context.Context, req ctrl.Request) (ctrl.Result, error) {
	l := log.FromContext(ctx)

	// OBSERVE desired — the ReplicaPin
	var pin opsv1.ReplicaPin
	if err := r.Get(ctx, req.NamespacedName, &pin); err != nil {
		return ctrl.Result{}, client.IgnoreNotFound(err)
	}
	// OBSERVE actual — the target Deployment
	var dep appsv1.Deployment
	key := types.NamespacedName{Namespace: pin.Namespace, Name: pin.Spec.DeploymentName}
	if err := r.Get(ctx, key, &dep); err != nil {
		return ctrl.Result{RequeueAfter: 10 * time.Second}, client.IgnoreNotFound(err)
	}
	// COMPARE + ACT
	want := pin.Spec.Replicas
	if dep.Spec.Replicas == nil || *dep.Spec.Replicas != want {
		l.Info("reconcile: pinning replicas", "deployment", dep.Name, "to", want)
		dep.Spec.Replicas = &want
		if err := r.Update(ctx, &dep); err != nil {
			return ctrl.Result{}, err
		}
	}
	return ctrl.Result{RequeueAfter: 10 * time.Second}, nil   // level-triggered resync
}

Add an RBAC marker so the controller may scale Deployments, then regenerate:

// +kubebuilder:rbac:groups=apps,resources=deployments,verbs=get;list;watch;update;patch
make generate && make manifests   # markers → CRD schema + RBAC Role

A1.4 · Install, run, test

make install                       # register the CRD in the cluster
make run                           # start the controller (terminal 1)

# terminal 2 — create the custom resource:
kubectl apply -f - <<'EOF'
apiVersion: ops.quarx.co/v1
kind: ReplicaPin
metadata: {name: pin-web, namespace: default}
spec: {deploymentName: web, replicas: 3}
EOF

# drift test — healed by the RequeueAfter timer (~10s): we only WATCH the ReplicaPin
kubectl scale deploy web --replicas=1
# desired-change test — healed INSTANTLY: a watch event on the ReplicaPin
kubectl patch replicapin pin-web --type=merge -p '{"spec":{"replicas":5}}'
Watch vs. resync — the key lesson

Changing the watched object (the ReplicaPin) reconciles instantly; drift on an unwatched object (scaling the Deployment) waits for the timer. Production upgrade: add .Owns(&appsv1.Deployment{}) in SetupWithManager to watch the Deployment too, making both instant.

Gotchas

  • Do dev work in the guest home ~ (/home/…guest). The /Users/… mount is read-only — curl -o and go build fail there.
  • API group apps is built-in and collides — use a custom group (ops).
  • kubebuilder needs make + build-essential; a minimal Ubuntu image ships neither.
  • Run kubectl from cp — the worker has no kubeconfig (localhost:8080 error = no config).
  • Edit the Spec → always make generate && make manifests to regenerate DeepCopy + CRD.

Decision framework — controller vs alternatives

Use a controller/operator

When custom state must be continuously reconciled — self-healing, drift correction, lifecycle management of your own resource type.

Use Helm / Kustomize / Job

Helm/Kustomize for one-shot templated installs; Job/CronJob for finite tasks. No ongoing loop.

Rule: if it needs to keep being true, it's a controller. Full lab in the repo: architect-track/a1-controller-pattern/.
